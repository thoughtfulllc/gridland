## Configure the Vite Plugin

Polyterm provides a Vite plugin that handles module resolution for the
underlying opentui engine. Add it to your `vite.config.ts`:

```ts title="vite.config.ts"
import { defineConfig } from "vite"
import react from "@vitejs/plugin-react"
import { polytermWebPlugin } from "@polyterm.io/web/vite-plugin"

export default defineConfig({
  plugins: [
    react(),
    ...polytermWebPlugin({
      opentuiPath: "../path/to/opentui",
    }),
  ],
})
```

<Callout type="warn">
The `opentuiPath` option must point to the root of the opentui monorepo so the
plugin can resolve `@opentui/core`, `@opentui/react`, and `@opentui/ui`.
</Callout>

## Create Your Component

Wrap your TUI content in `TUI`:

```tsx title="App.tsx"
import { TUI } from "@polyterm.io/web"

function TuiDemo() {
  return (
    <box flexDirection="column" padding={1}>
      <box border borderStyle="rounded" borderColor="#88c0d0" padding={1}>
        <text fg="#a3be8c">Hello from Vite!</text>
      </box>
    </box>
  )
}

export default function App() {
  return (
    <TUI style={{ width: "100vw", height: "100vh" }}>
      <TuiDemo />
    </TUI>
  )
}
```

## How It Works

The Vite plugin sets up:

- **Module aliases** — Redirects `@opentui/core` and Node.js built-ins to
  browser-safe shims
- **FFI stubs** — Replaces Zig/Bun FFI imports with JavaScript implementations
- **npm redirects** — Virtual modules for CJS-to-ESM conversion of packages
  imported by the opentui monorepo
- **Dependency pre-bundling** — Ensures `react`, `react-reconciler`,
  `yoga-layout`, and other dependencies are optimized

## Interaction Support

Polyterm supports user interaction on the canvas:

- **Text selection** — Click and drag to select text
- **Copy** — Selected text can be copied to clipboard
- **Paste** — Clipboard content can be pasted into input components
- **Drag and drop** — Files can be dropped onto the canvas
- **Keyboard** — Key events are forwarded to focused components
