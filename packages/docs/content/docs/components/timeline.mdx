The `Timeline` component renders a collapsible list of steps with animated spinners,
status indicators, and optional output text. Useful for showing AI reasoning chains,
build pipelines, or any multi-step process.

## Usage

```tsx title="Basic (collapsed)"
import { Timeline } from "@gridland/ui"

<Timeline
  steps={[
    { tool: "Read", label: "Reading files", status: "done" },
    { tool: "Edit", label: "Applying changes", status: "done" },
  ]}
  duration="3.2s"
  collapsed
/>
```

### Custom header label

```tsx title="Custom label"
<Timeline
  steps={[
    { tool: "Install", label: "Installing dependencies", status: "done" },
    { tool: "Build", label: "Compiling TypeScript", status: "running" },
    { tool: "Test", label: "Running test suite", status: "pending" },
  ]}
  duration="12s"
  collapsed={false}
  headerLabel="Build pipeline"
/>
```

## Props

### Timeline

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `steps` | `Step[]` | — | Array of step entries to display |
| `duration` | `string` | `"0ms"` | Duration string shown in the header |
| `collapsed` | `boolean` | `true` | Whether the step list is hidden |
| `headerLabel` | `string` | `"Thought for"` | Label prefix shown before the duration |

### Step

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `tool` | `string` | — | Tool or operation name (e.g. "Read", "Edit") |
| `label` | `string` | — | Description of the step |
| `status` | `"done" \| "running" \| "pending" \| "error"` | — | Current step status |
| `duration` | `string` | — | Optional duration for the individual step |
| `output` | `string` | — | Output or detail text shown below the step |
