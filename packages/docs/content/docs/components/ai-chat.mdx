The `AIChat` component from `@gridland/ui` renders a complete terminal-style AI chat
interface. It composes `Message`, `Timeline`, `PromptInput`, and `StatusBar` into
a ready-to-use chat view. It handles SSE streaming from an LLM API, displays
reasoning steps via `Timeline`, and includes slash commands and model switching
via `PromptInput`.

## Usage

```tsx
import { AIChat } from "@gridland/ui"
import { useKeyboard } from "@opentui/react"

function App() {
  return (
    <AIChat
      useKeyboard={useKeyboard}
      apiUrl="/api/chat"
    />
  )
}
```

## Architecture

`AIChat` composes these Gridland primitives:

| Component | Role |
|-----------|------|
| [`Message`](/docs/components/chat-bubble) | Renders each message with role-based styling, streaming cursor |
| [`Message.Reasoning`](/docs/components/chat-bubble) | Shows reasoning steps via `Timeline` |
| [`PromptInput`](/docs/components/prompt-input) | Input with slash commands, history, submit/stop |
| [`StatusBar`](/docs/components/status-bar) | Bottom bar showing model, keybindings |

## Features

### Streaming

- Connects to an SSE endpoint and parses `step_start`, `step_done`, `text`, `error`, and `done` events
- Typewriter effect reveals text 3 characters at a time at ~60fps
- Streaming cursor shown via `Message.Text isLast` during generation

### Reasoning Steps

During streaming, reasoning steps are rendered via `Message.Reasoning` â†’ `Timeline`:

- **Collapsed** (default): Shows header only ("Thought for 1.2s")
- **Expanded**: Full step timeline with animated spinners, status dots, and detail text

Toggle with `Ctrl+Shift+E`.

### Slash Commands

Handled by `PromptInput`:

- `/help` â€” Show available commands
- `/clear` â€” Clear the conversation
- `/model` â€” Switch between models (sonnet, haiku, opus)

### Input History

Handled by `PromptInput` â€” use up/down arrows to recall previously submitted messages.

### AIChatModel

```tsx

}
```

### AIChatMessage

```tsx
interface AIChatMessage {
  role: "user" | "assistant"
  text: string
  steps?: AIChatStep[]
}
```

### AIChatStep

```tsx
interface AIChatStep {
  tool: string
  label: string
  duration: string
  status: "done" | "running" | "pending"
  detail?: string
}
```

## SSE Event Format

The API endpoint should stream Server-Sent Events with these event types:

| Event | Payload | Description |
|-------|---------|-------------|
| `step_start` | `{ type, tool, label }` | A reasoning step has started |
| `step_done` | `{ type, tool, label, duration, detail }` | A step completed |
| `text` | `{ type, text }` | Append text to the response |
| `error` | `{ type, error }` | An error occurred |
| `done` | `{ type }` | Stream complete |

## Keybindings

Most keybindings are handled by `PromptInput`. AIChat adds:

| Key | Action |
|-----|--------|
| `Ctrl+Shift+E` | Toggle reasoning step timeline |
