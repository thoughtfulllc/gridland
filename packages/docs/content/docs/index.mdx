OpenTUI Web renders terminal-style UIs directly to an HTML5 Canvas using React.
No xterm.js, no terminal emulator — just a React reconciler, Yoga layout, and canvas pixels.

## Features

- **Canvas rendering** — TUI content drawn directly to `<canvas>`, not a DOM tree
- **React reconciler** — Use JSX with `<box>`, `<text>`, and other TUI primitives
- **SSR-safe** — Single `<OpenTuiCanvas>` component with built-in client detection
- **Yoga layout** — Flexbox positioning via `yoga-layout`
- **Framework support** — Works with Vite and Next.js
- **Interaction** — Text selection, copy/paste, keyboard events, and drag-and-drop

## Quick Start

Install the package in your project:

<Tabs items={['npm', 'yarn', 'pnpm', 'bun']}>
<Tab value="npm">
```bash title="Terminal"
npm install opentui-web
```
</Tab>
<Tab value="yarn">
```bash title="Terminal"
yarn add opentui-web
```
</Tab>
<Tab value="pnpm">
```bash title="Terminal"
pnpm add opentui-web
```
</Tab>
<Tab value="bun">
```bash title="Terminal"
bun add opentui-web
```
</Tab>
</Tabs>

Then wrap your TUI content in `OpenTuiCanvas`:

```tsx title="App.tsx"
import { OpenTuiCanvas } from "opentui-web"

function App() {
  return (
    <OpenTuiCanvas style={{ width: "100vw", height: "100vh" }}>
      <box border borderStyle="rounded" padding={1}>
        <text fg="#a3be8c">Hello from OpenTUI Web!</text>
      </box>
    </OpenTuiCanvas>
  )
}
```

<Callout type="info">
You'll also need to configure your bundler (Vite or webpack) to resolve OpenTUI's internal modules. See the framework guides below.
</Callout>
