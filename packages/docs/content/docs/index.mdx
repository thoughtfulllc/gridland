Gridland renders terminal-style UIs directly to an HTML5 Canvas using React.
Built on the [opentui](https://opentui.com) engine. No xterm.js, no terminal emulator, just a React reconciler, Yoga layout, and canvas pixels.

## Start a new project

Scaffold a new Gridland app with everything pre-configured:

<Tabs items={['bun', 'npm', 'yarn', 'pnpm']}>
<Tab value="bun">
```bash title="Terminal"
bunx create-gridland my-app
```
</Tab>
<Tab value="npm">

</Tab>
<Tab value="yarn">

</Tab>
<Tab value="pnpm">

</Tab>
</Tabs>

## Install components

Gridland UI components are distributed via a [shadcn](https://ui.shadcn.com) registry. Install individual components into your project so you own the code and can customize it freely.

If you scaffolded with `create-gridland`, shadcn is already configured — just run:

```bash
bunx create-gridland add spinner
```

`create-gridland add` detects your package manager and proxies to shadcn under the hood. You can install multiple components at once (`bunx create-gridland add modal side-nav`) and preview the resolved command with `--dry-run`.

Otherwise, configure shadcn in your project and call it directly:

```bash
bunx shadcn@latest add @gridland/spinner
```

See the [Installing Components](/docs/guides/components) guide for setup instructions and the full list of available components.
